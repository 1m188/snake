这是一个用 Python 编写的运行于终端中的贪吃蛇小游戏。

## 功能需求
- 蛇
    - 初始的时候蛇一共只有三节，方向任意，初始化到场地中央，每次吃到场地中的食物就能够增长一节，不停的在移动，一旦蛇头撞到了“墙”（边界）或者自己的身体，那么游戏结束。
    - 蛇使用 '#' 符号作为身体， '@' 符号作为蛇头。

- 食物
    - 食物在最开始随机初始化到场地中任一坐标位置处，直到被蛇吃掉，然后再次在任一坐标处随机生成。
    - 游戏有计分机制，每次吃到一个食物计1分，分数显示在右上角。
    - 食物采用 '*' 符号表示。

- 按键
    - 使用wasd进行移动。按下（不必等到释放）按键即刻改变方向。
    - 当敲击一次空格之后，蛇的移动速度变为原来的三倍；当再次敲击空格的时候，蛇的移动速度还原。
    - 在游戏中不接受其他任何按键。当进入到游戏结束界面的时候，按下esc退出游戏。

- 地图
    - 整个游戏界面撑满整个终端，并且在地图边界处要有 '-' '|' '+' 等符号所绘制的地图边界。

- 游戏结束界面
    - 在界面中央显示 “游戏结束！” 文本。
    - 在界面中央显示最终得分。
    - 显示 “按下任意键继续” 的文本。当按下任意键后重新开始新的一局游戏。
    - 显示 “按下esc键退出游戏” 的文本。当按下esc键后退出游戏，此时不再开始新的一局游戏。
    - 上述文本全部居中对齐，每一条占据一行（或者两行），保持整个界面的匀称、饱满和几何美感。

- 其他
    - 游戏能够兼容不同的平台和终端，win、Linux、macos，powershell、cmd、bash、或者vscode的集成终端中，它都要能够良好的运行。
    - 游戏采用双缓冲策略渲染，也即在游戏循环中，每次在内存中计算好一帧的画面（包括蛇和食物等各种对象的状态）再整体渲染（绘制）到终端上，以避免画面撕裂或者闪烁等问题。
    - 采用 “帧” 的概念，也即每秒钟渲染一定数目的画面，每个画面称为 帧 。结合上一条，当一帧被渲染到终端上之后，要等待固定的时间再绘制渲染下一帧，中间的这段时间可以用来进行画面的计算。游戏采取60帧的设置。

## 代码要求
- 代码尽可能简单直接，不要为了不存在的扩展性或者伸缩性添加额外的复杂度，不要添加任何冗余的结构或者内容。

- 采取模块化开发方式，架构清晰易懂，便于后续维护，做到模块内部高聚合，模块之间低耦合。

- 代码保持有良好的风格，添加周全详细的注释文档以强化其可读性。每个代码文件的顶部应该采用段注释说明该文件的结构和主要内容，起到一个文件摘要的作用。所有的注释文档应该随着代码的迭代而同步更新，避免出现文档落后的情况。

- 代码要进行格式化以保持风格一致。本项目中 Python 代码的风格应该遵守 black 的默认格式要求。
